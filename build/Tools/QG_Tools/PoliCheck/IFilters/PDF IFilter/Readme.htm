Adobe ® PDF iFilter 6.0 for Windows Read Me

October 1, 2004

 

Welcome to the Adobe® PDF iFilter 6.0 Read Me.  PDF iFilter is a free downloadable application which lets you index PDF documents with Microsoft® indexing clients, so that users can easily search for text within PDF documents.  Adobe PDF iFilter 6.0 is an update to Adobe PDF iFilter 5.0 and includes the following improvements:

·         Support for all versions of PDF files including PDF 1.5

·         Support for additional operating systems (see System Requirements below)

 

This Read Me contains information on:

·         System Requirements

·         Installation Instructions

·         Troubleshooting

·         Known Issues

·         Additional Information

 

System Requirements

 

PDF iFilter 6.0 requires one of the following environments:

·         Microsoft Windows 2000 Professional, Service Pack 2

·         Microsoft Windows XP Professional, Service Pack 1

·         Microsoft Windows 2000 Server, Service Pack 3

·         Microsoft Windows 2003 Server

Each of these environments must also contain Microsoft Indexing Services.

 

In addition, PDF iFilter 6.0 has been tested in the following environments:

·         Microsoft Windows 2000 Server (Service Pack 3) with Microsoft Office SharePoint Portal Server 2001

·         Microsoft Windows 2003 Server with Microsoft Office SharePoint Portal Server 2003

 

Installation Instructions

 

1.      Download PDF iFilter from http://www.adobe.com onto the machine where you wish to install it.

2.      Stop all appropriate clients, using one or more of the following methods:

·         Windows XP and 2003 Server:  Go to Start > Control Panel > Administrative Tools > Computer Management.  In left-side Console Tree, highlight Indexing Service under Services and Applications.  Under Action Menu, choose Stop.  Close window.

·         Windows 2000 and 2000 Server:  Go to Start > Settings > Control Panel > Administrative Tools > Computer Management.  In left-side Console Tree, highlight Indexing Service under Services and Applications.  Under Action Menu, choose Stop.  Close window.

·         SharePoint 2001 and 2003:  Go to Start > Settings > Control Panel > Administrative Tools > Services.  Right-click on SharePoint Portal Server and choose Stop.

3.      Uninstall any previous version of PDF iFilter.

4.      Double-click the downloaded PDF iFilter 6.0 EXE file and follow the on-screen instructions.

5.      After installation completes, start all appropriate clients, using one or more of the following methods:

·         Windows XP and 2003 Server:  Go to Start > Control Panel > Administrative Tools > Computer Management.  In left-side Console Tree, highlight Indexing Service under Services and Applications.  Under Action Menu, choose Start.  Close window.

·         Windows 2000 and 2000 Server:  Go to Start > Settings > Control Panel > Administrative Tools > Computer Management.  In left-side Console Tree, highlight Indexing Service under Services and Applications.  Under Action Menu, choose Start.  Close window.

·         SharePoint 2001 and 2003:  Go to Start > Settings > Control Panel > Administrative Tools > Services.  Right-click on SharePoint Portal Server and choose Start.

6.      [SHAREPOINT ONLY]  Add PDF as a file type to be included in the content index.  On the SharePoint Site Settings Page, in the Search Settings and Indexed Content section, chose Configure search and indexing.  On the Configure Search and Indexing page, in the General Content Settings and Indexing Status, choose Include File Types.  On the Include File Types page, choose Add File Type.  On the Add File Type page, in the File Extension box, type PDF for the file type that you want to add to the list of file types for the content index.  Choose OK.

7.      Re-index your system with the appropriate clients, using one or more of the following methods:

·         Windows 2000/2000 Server/XP/2003 Server:  Open the Computer Management Console (mentioned above).  In the left-side Console Tree, open Services and Applications > Indexing Service.  Open the first Catalog listed under Indexing Service by clicking on the “+” sign next to the Catalog name.  Click on Directories.  For each directory listed in the right-hand pane which contains PDF files, right-click on it and select All Tasks > Rescan (Full).  If you have additional Catalogs listed under Indexing Service, repeat these steps.

·         SharePoint 2001 and 2003:  On the SharePoint Site Settings Page, in the Manage Search Settings and Indexed Content section, chose Configure search and indexing.  On the Configure Search and Indexing page, in the Content Indexes section, choose Manage the list of indexes.  On the Manage Content Indexes page, rest the cursor on the index name and click the arrow which appears.  On the menu which appears, click Full Update.

 

Troubleshooting

 

If your search method does not find text in a PDF file after you install the PDF iFilter and after you re-index your site:

·         Restart your computer

·         Verify that the PDF file is not damaged or password protected. The PDF iFilter cannot index PDF files that cannot be opened in Acrobat. Open the PDF file in Adobe Reader 6, Adobe Acrobat 6 Standard or Adobe Acrobat 6 Professional to verify that the file is not damaged or password protected.

·         Verify that the PDF file does not contain Security Permissions prohibiting you from searching the file.  Open the file and choose File > Document Properties > Security.  Verify that the value for "Content Copying or Extraction" is "Allowed". If it is "Not Allowed", the PDF iFilter cannot index the text.

·         Verify that the PDF file contains searchable text. You can verify if there is searchable text in the PDF file by opening it in Reader or Acrobat and selecting the text with the Text Select tool.  Or, you can open the file and using the Edit > Select All command to select text.  If neither method highlights any text, it is likely that the PDF contains a source image of text which the PDF iFilter cannot index.  You can then use a tool like Acrobat’s Paper Capture tool (Document > Paper Capture > Start Capture…) to convert the source image of text into searchable text which PDF iFilter can index.

·         Verify that the PDF file contains text that is properly encoded.  If the file contains searchable text, yet the Acrobat Search tool (Edit > Search) cannot find the text, then the text may not be properly encoded.  Select a word using the Text Select tool, copy it to your clipboard, and paste it into a text editor (e.g. Notepad).  If the word is not legible in the text editor, the file contains text or fonts that are not properly encoded.

·         Ensure the PDF file has a ".pdf" filename extension. Clients, such as Index Services, find the PDF iFilter by looking up the filename's extension in the Windows Registry.

·         If the original document was created in a Windows 95 application and uses TrueType fonts, either reformat the text with Type 1 fonts or use Adobe PDF Printer to recreate the PDF file. In Windows 95, PDF files generated from PostScript files containing TrueType fonts include text that is not searchable.  Both the Microsoft and Adobe PostScript® printer drivers remap TrueType fonts at PostScript generation time, which disconnects the character you see on screen from that character in the font (for more information, see http://www.adobe.com/support/techdocs/12f6e.htm ).

 

Known Issues

 

In some cases, when subsetted (embedded) fonts are used in a PDF file, words that are viewable in Acrobat cannot be searched using Index Services.

PDF iFilter will not extract text from PDF files which are password protected.  Password-protected PDF files will not appear on an "unfiltered files" list.

PDF iFilter will not extract text from PDF files which have protection against copying.  This protection is set when the PDF file has Acrobat Standard Security and "No Content Copying or Extraction…..” selected.

The Highlight Hits feature in Microsoft Index Services cannot highlight text in a PDF file opened in Acrobat.  The iFilter specification does not provide any way to return information about how to highlight text in an external application. It only provides the means to extract text from a file of a given type. As a workaround, use the Search feature in Acrobat or Reader to find a specific text strings in a PDF file.

The PDF iFilter DLL is not reentrant (i.e., you cannot run multiple copies of it at the same time).

 

Additional Information

 

Microsoft Developer Support

An iFilter is a DLL (Dynamic Link Library) file that is called by a client to extract words from a given file format.  The Adobe PDF iFilter consists of code that understands the PDF file format along with code that provides the appropriate interface to callers like Index Services.  The Adobe PDF iFilter is identified when a caller looks in the Windows Registry for a DLL file that knows how to handle files with extension ".pdf."  The Adobe PDF iFilter returns text to the caller, which is added to the catalog - or index - being built.

 

To obtain more information about the iFilter specification and writing custom iFilter methods, visit this location on Microsoft's web site:

http://msdn.microsoft.com/library/default.asp?url=/library/en-us/indexsrv/html/ixrefint_9sfm.asp

Language Support

Adobe PDF iFilter has no user interface and therefore is language-agnostic.  It is tested and supported in Tier 1 languages (English, French, German, and Japanese), utilizing operating systems in the above languages and text within Adobe PDF files in the above languages.  However it has also undergone additional limited testing utilizing Adobe PDF documents in all Acrobat languages (see http://www.adobe.com/products/acrobat/acrrlanguages.html) and has correctly returned text to Microsoft Indexing Services.  Since the Adobe PDF iFilter is based on the Microsoft indexing client, it is this indexing client which is responsible for interpreting the returned text and then presenting the information to the user.

 

Searching by Metadata

The standard metadata in a PDF file is Title, Subject, Author and Keywords.  You may search for this metadata by issuing a query like:

 

            @docTitle title

            @docSubject subject

            @docAuthor author

            @docKeywords keyword

 

By default, the Index Services only caches @docTitle.  For improved performance, you may wish to turn on caching for the @docAuthor, @docSubject and @docKeywords.  For example, in the Computer Management console (mentioned above), open Services and Applications > Indexing Service.  Open the first Catalog listed under Indexing Service by clicking on the “+” sign next to the Catalog name.  Click on Properties.  In the right-hand pane, right-click on the desired property (for example, use the Friendly Name "DocAuthor"), choose Properties and then click the Cached checkbox in the dialog box that opens (using the VT_LPWSTR data type).  If you have additional Catalogs listed under Indexing Service, repeat these steps.
